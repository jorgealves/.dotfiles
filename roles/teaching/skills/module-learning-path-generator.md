## Purpose
Design end-to-end learning paths that take students from foundational concepts to job-ready skills, aligned with specific career tracks (backend engineer, data engineer, DevOps, etc.).

## When to Use
- User asks to design a course, curriculum, or multi-week program
- User needs to structure a learning track for a specific role or technology
- User wants to map prerequisites and skill dependencies across modules

## What It Does
1. Identifies the target role/outcome and required skill level
2. Breaks the path into modules with clear learning objectives per module
3. Sequences modules by dependency (no knowledge gaps)
4. Assigns estimated duration and project milestones
5. Maps each module to real-world job tasks

## Inputs
- Target career role or technology focus
- Student starting level (beginner / intermediate / advanced)
- Available time (e.g., 8 weeks, 3 months)

## Output
- Structured learning path with numbered modules
- Per-module: objectives, topics, exercises, and capstone project
- Prerequisite map
- Success criteria per module
