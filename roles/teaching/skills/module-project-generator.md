## Purpose
Create professional-grade project briefs that give students realistic, portfolio-worthy tasks aligned to specific learning module objectives.

## When to Use
- User needs a project assignment for a specific module or topic
- User wants students to apply concepts in a realistic context
- User is designing a capstone or assessment project

## What It Does
1. Maps project requirements to learning objectives
2. Designs a realistic problem domain (not toy examples)
3. Specifies deliverables, acceptance criteria, and grading rubric
4. Includes stretch goals for advanced students
5. Provides starter code or scaffolding guidance

## Inputs
- Learning objectives to reinforce
- Technology stack
- Complexity level and time budget for students

## Output
- Full project brief with context and requirements
- User stories / acceptance criteria
- Technical constraints and scaffolding hints
- Grading rubric (conceptual mastery + implementation)
- Extension challenges
